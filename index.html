AI Follow-Up Automation

Automated email/SMS follow-up sequence built with Make/Zapier to re-engage cold leads without manual effort.

Automation Flow

  1. Trigger: Lead tagged "Cold" in CRM (no reply in 14+ days)
  2. Day 0: Send re-engagement email
  3. Wait 2 days → if no open/click → Day 2: Send SMS
  4. Wait 3 days → if no reply → Day 5: Send final email
  5. If reply at any step → auto-remove from sequence, notify rep

Email 1 — Day 0

Subject: Still thinking about [Service/Offer]?
Hi [First Name], I noticed we hadn't connected in a bit — wanted to check if [pain point] is still something you're looking to solve. If now's a better time to chat, I'd love to grab 15 minutes this week. Here's my calendar: [Booking Link]. No pressure either way — just here when you're ready. — Kees

SMS — Day 2

Hi [First Name], it's Kees 👋 Just following up on my last email — would a quick 15-min call this week work for you? Here's my calendar: [Booking Link]

Email 3 — Day 5 (Final)

Subject: Closing the loop
Hi [First Name], didn't want to keep filling your inbox, so this'll be my last note for now. If priorities shift and you'd like to revisit [Service/Offer], I'm just an email away. Wishing you the best either way! — Kees

Process Audit Sample

A blind QA audit workbook sampling a minimum of 3% of each Actor ID's resolved tickets per queue — no agent names, identity-blind by design. Agent decisions are pulled via VLOOKUP from a Scuba database export (TSV), with accuracy automatically flagged whenever the QA reviewer's decision differs from the agent's.

Included Sheets

  • Raw Export — simulated Scuba TSV export of resolved tickets across all queues, keyed by Actor ID
  • QA Audit (Demo Sample) — main queues, VLOOKUP-driven review with auto Match/ERROR flagging
  • QA Manual Audit Template — select an Actor ID and the Ticket ID, Queue, Date & Agent Decision auto-populate; enter your QA Decision & Notes to auto-flag accuracy
  • Escalation Tagging Audit — separate table, rated GOOD / FAIR / BAD instead of a violation category, with its own accuracy-by-actor breakdown
  • Support Investigation Audit — separate table using the same violation categories, with its own accuracy-by-actor breakdown
  • Accuracy Summary — accuracy % and audit coverage % per Actor and per Queue for the main queues (overall queue-level rollup included)

Works in Google Sheets too

Open Google Drive → New → File upload → select the downloaded file → it auto-converts, or open it and choose "Open with Google Sheets." All formulas and dropdowns carry over.
